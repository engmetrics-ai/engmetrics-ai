# Community

Welcome to the EngMetrics AI community. This page explains how to get involved, ask for help, and contribute to the project.

---

## How to get involved

### Use the tool

The fastest way to contribute is to use the tool on your own team and share what you find. Running in mock mode requires zero setup:

```bash
pipx install ai-engineering-metrics   # or: uvx ai-engineering-metrics --help
ai-engineering-metrics analyze --mock
```

Open the generated dashboard and notice what works well and what doesn't. That feedback is valuable.

### File issues

Found a bug? Have a feature idea? Missing documentation? Open a [GitHub issue](https://github.com/engmetrics-ai/engmetrics-ai/issues). Use the templates to make it easy for maintainers to triage and respond.

- **Bug report** — for something that is broken.
- **Feature request** — for something you'd like to see.
- **Documentation improvement** — for missing, unclear or incorrect docs.

### Join discussions

[GitHub Discussions](https://github.com/engmetrics-ai/engmetrics-ai/discussions) is the right place for open-ended questions, ideas, and conversations that don't fit neatly into a bug or feature request.

### Contribute code or docs

See [CONTRIBUTING.md](../CONTRIBUTING.md) for the development setup and workflow. `good first issue` tickets are a great starting point.

---

## Good first issues

Look for the [`good first issue`](https://github.com/engmetrics-ai/engmetrics-ai/labels/good%20first%20issue) label for contributions that are well-scoped and don't require deep knowledge of the codebase.

Examples of good first issues:
- Adding or improving documentation
- Adding tests for untested edge cases
- Improving CLI error messages or help text
- Adding a new environment variable example to `.env.example`

---

## Communication norms

- Be kind and constructive. See the [Code of Conduct](../CODE_OF_CONDUCT.md).
- Assume good faith — most questions and suggestions come from genuine curiosity or need.
- Keep issues focused. One bug or feature per issue.
- Link to related issues or PRs to give context.
- For security issues, do **not** open a public issue — see [SECURITY.md](../SECURITY.md).

---

## Recognition

All contributors are credited in the repository. First-time contributors are especially welcome — this project is explicitly designed for newcomers to open source.

---

## Maintainer response times

This is a small project maintained by one person with a day job. Realistic expectations:

- **Issues:** triaged within 1 week.
- **PRs:** reviewed within 2 weeks (usually faster for small, focused PRs).
- **Discussions:** best-effort.

If your PR is stuck, feel free to ping in the discussion thread after 2 weeks.
